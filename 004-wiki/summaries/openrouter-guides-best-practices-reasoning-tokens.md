# Openrouter Guides Best Practices Reasoning Tokens

## Key Points

- OpenRouter's `reasoning` parameter provides a unified interface to control reasoning tokens across providers, normalizing effort levels (OpenAI-style) and `max_tokens` (Anthropic-style) into a single config object ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- Reasoning tokens are output tokens that expose model reasoning steps; they are included by default and charged as output tokens ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- Five effort levels control reasoning allocation: `xhigh` (~95%), `high` (~80%), `medium` (~50%), `low` (~20%), `minimal` (~10%), and `none` (disables reasoning entirely) ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- The `exclude` flag lets models reason internally without returning reasoning tokens in the response; `enabled: true` activates reasoning at medium effort with no exclusions ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- Reasoning can be preserved across multi-turn conversations by passing `reasoning` (plaintext) or `reasoning_details` (structured array) back in assistant messages; critical for tool calling continuity ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- The `reasoning_details` API returns three types: `reasoning.summary`, `reasoning.encrypted`, and `reasoning.text`; formats include `anthropic-claude-v1`, `openai-responses-v1`, `google-gemini-v1`, `xai-responses-v1`, and `azure-openai-responses-v1` ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- Provider-specific implementations: Anthropic supports `reasoning.max_tokens` (1,024–128,000 range); Gemini 3 uses `thinkingLevel` mapped from effort; OpenAI/Grok use effort levels directly; some Alibaba Qwen models map `max_tokens` to `thinking_budget` ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]

## Quotes

- "Reasoning tokens provide a transparent look into the reasoning steps taken by a model. Reasoning tokens are considered output tokens and charged accordingly." ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md:5]
- "Some reasoning models do not return their reasoning tokens — While most models and providers make reasoning tokens available in the response, some (like the OpenAI o-series) do not." ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md:9]
- "When providing reasoning_details blocks, the entire sequence of consecutive reasoning blocks must match the outputs generated by the model during the original request; you cannot rearrange or modify the sequence of these blocks." ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md:234]
- "`max_tokens` must be strictly higher than the reasoning budget to ensure there are tokens available for the final response after thinking." ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md:291]

## Notes

- Legacy parameters `include_reasoning: true` and `include_reasoning: false` are still supported but deprecated in favor of the unified `reasoning` parameter ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- The `:thinking` variant is no longer supported for Anthropic models; use the `reasoning` parameter instead ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- `reasoning_content` is an alias for `reasoning` when preserving reasoning context; both function identically ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
- Z.ai preserved thinking (as distinct from standard interleaved thinking) is currently not supported for reasoning preservation ^[raw/document/openrouter/openrouter-072-guides-best-practices-reasoning-tokens-2026-04-29.md]
