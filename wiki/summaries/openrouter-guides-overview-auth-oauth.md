# OpenRouter OAuth PKCE Authentication

## Key Points

- Users connect to OpenRouter via OAuth PKCE (Proof Key for Code Exchange) in a three-step flow: redirect to `/auth`, exchange the authorization code for an API key, then use the key for API requests ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]
- The `code_challenge_method` parameter supports `S256` (recommended, SHA-256-based) and `plain`; `code_challenge` itself is optional but recommended for security ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]
- The S256 code challenge is generated by hashing a random `code_verifier` string with SHA-256 and base64url-encoding the result, using the Web Crypto API and Buffer API ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]
- After user authorization, the authorization code is exchanged at `POST /api/v1/auth/keys` by sending `code`, `code_verifier`, and `code_challenge_method`; the response returns a `key` field containing the user-controlled API key ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]
- Localhost apps should use `http://localhost:3000` as the callback and referrer URL during development, then switch to a public GitHub repo or project website URL for production ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]
- Error codes: `400 Invalid code_challenge_method` (mismatch between steps), `403 Invalid code or code_verifier` (user not logged in or wrong verifier), `405 Method Not Allowed` (must use POST and HTTPS) ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]

## Quotes

- "For maximum security, set `code_challenge_method` to `S256`, and set `code_challenge` to the base64 encoding of the sha256 hash of `code_verifier`." ^[raw/document/openrouter/openrouter-012-guides-overview-auth-oauth-2026-04-29.md]

## Notes

- The PKCE flow produces a user-controlled API key, meaning each end user's usage is billed to their own OpenRouter account rather than the app developer's account
- The resulting API key works with both the OpenRouter TypeScript SDK (`@openrouter/sdk`) and direct fetch calls to the chat completions endpoint
